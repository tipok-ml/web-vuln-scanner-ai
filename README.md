# Web Vulnerability Scanner + AI Report

[![Python 3.10+](https://img.shields.io/badge/python-3.10%2B-blue)](https://www.python.org/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![DeepSeek](https://img.shields.io/badge/AI-DeepSeek-green)](https://deepseek.com)

**Proof‑of‑Concept** — асинхронный сканер веб-уязвимостей (SQLi, XSS, Blind SQLi) с генерацией отчёта через DeepSeek API.  
Проект демонстрирует навыки парсинга HTML, работы с HTTP, асинхронного краулинга и интеграции с языковой моделью.

> ⚠️ **Важно:** Это демонстрационный инструмент, **не предназначенный для аудита реальных сайтов**.  
> В текущей версии используется примитивная логика детекта (поиск сигнатур в ответе), что приводит к ложным срабатываниям и пропуску многих уязвимостей. Для профессионального тестирования используйте `sqlmap`, `XSStrike`, `Burp Suite` или другие mature‑решения.

## Возможности

- **Краулинг** — рекурсивный обход страниц одного домена (макс. 20 страниц по умолчанию).
- **Парсинг форм** — извлечение HTML‑форм, `input`, `textarea`, `select`, а также GET‑параметров из URL.
- **SQL Injection** — обнаружение error‑based (по строкам ошибок) и time‑based blind (по задержке ответа).
- **Reflected XSS** — проверка отражения пейлоадов в теле ответа.
- **AI‑отчёт** — отправка результатов в DeepSeek для генерации структурированного отчёта с рекомендациями на русском языке.
- **Экспорт** — сохранение отчёта в Markdown, JSON, HTML (по выбору).
- **Асинхронность** — использование `aiohttp` для параллельных запросов.
- **CLI** — гибкий запуск через аргументы командной строки.

## Ограничения

| Аспект | Детали |
|--------|--------|
| **Детект SQLi** | Ищет 7 заранее заданных строк в ответе; обходится параметризованными запросами. |
| **Детект XSS** | Проверяет точное совпадение пейлоада; легко блокируется эскейпингом. |
| **Time‑based blind** | Порог в 4.5 секунды даёт ложные срабатывания на медленных серверах. |
| **Краулинг** | Ограничен одним доменом, не обрабатывает JavaScript‑рендеринг, `robots.txt`. |
| **AI‑отчёт** | Генерируется на основе данных ненадёжного сканера — возможны ложные рекомендации. |
| **Rate limiting** | Отсутствует — сканер может перегрузить целевой сервер. |

## Быстрый старт

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/tipok-ml/web-vuln-scanner-ai.git
cd web-vuln-scanner-ai
```

### 2. Создайте виртуальное окружение (рекомендуется)

```bash
python3 -m venv venv
source venv/bin/activate   # Linux/Mac
# venv\Scripts\activate    # Windows
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. (Опционально) Задайте API‑ключ DeepSeek

```bash
export DEEPSEEK_API_KEY="sk-ваш_ключ"
```

Без ключа AI‑отчёт будет пропущен.

### 5. Запустите локальный тестовый сервер (в отдельном терминале)

```bash
python test_server.py
```

Сервер запустится на `http://127.0.0.1:8000` и будет имитировать уязвимости.

### 6. Запустите сканер

```bash
python main.py -u http://127.0.0.1:8000 --depth 5 --concurrency 3 --formats md json -v
```

Вы увидите найденные уязвимости (SQLi + XSS) и, при наличии ключа, сгенерированный AI‑отчёт.

## Использование

```
python main.py [-h] -u URL [--depth DEPTH] [--concurrency CONCURRENCY]
               [--timeout TIMEOUT] [--output OUTPUT] [--formats {md,json,html} [{md,json,html} ...]]
               [--no-ai] [-v]
```

Обязательные аргументы:

| Аргумент | Описание |
|----------|----------|
| `-u, --url` | Целевой URL (например, `http://example.com`) |

Опциональные аргументы:

| Аргумент | По умолчанию | Описание |
|----------|--------------|----------|
| `--depth` | 20 | Максимальное количество сканируемых страниц |
| `--concurrency` | 5 | Количество параллельных запросов |
| `--timeout` | 10 | Таймаут HTTP‑запроса (сек) |
| `--output` | `report` | Базовое имя файлов для отчётов |
| `--formats` | `md json` | Форматы отчёта (`md`, `json`, `html`) |
| `--no-ai` | – | Пропустить генерацию AI‑отчёта |
| `-v` | – | Подробный вывод (debug) |

### Примеры

```bash
# Сканирование локального стенда
python main.py -u http://127.0.0.1:8000 -v

# Сканирование внешнего сайта (только с разрешения владельца)
python main.py -u http://testphp.vulnweb.com --depth 10 --concurrency 2 --formats md html

# Без AI, экспорт только JSON
python main.py -u http://127.0.0.1:8000 --no-ai --formats json
```

## Генерация AI‑отчёта

После сканирования найденные уязвимости отправляются в DeepSeek (модель `deepseek-chat`) с промптом:

```
Ты – эксперт по кибербезопасности. Проанализируй результаты...
```

DeepSeek возвращает структурированный отчёт на русском языке, который сохраняется в выбранные форматы.

**Важно:** Качество AI‑отчёта напрямую зависит от качества данных, переданных сканером. В текущей версии возможны ложные рекомендации.

## Безопасность и этика

- **Сканируйте только те сайты, на которые у вас есть письменное разрешение.**
- Несанкционированное сканирование может нарушать законодательство (УК РФ ст. 272, 273, 274 и аналоги в других странах).
- В текущей версии отсутствуют механизмы rate limiting — сканер может создать избыточную нагрузку на сервер.
- API‑ключ DeepSeek храните в переменной окружения, **не добавляйте его в код**.
